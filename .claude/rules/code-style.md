# Code Style & Naming Rules

## Formatting
- Use Prettier for consistent formatting.
- Indentation: 2 spaces.
- Semicolons: Required.
- Single quotes preferred for strings.

## Naming Conventions
- **Files**: `PascalCase` for React components, `kebab-case` for others.
- **Variables/Functions**: `camelCase`.
- **Constants**: `UPPER_SNAKE_CASE`.
- **Folders**: `kebab-case`.

## Imports
- Group imports: React, External libs, Internal components, Hooks, Styles.
- Avoid deep relative paths (use aliases like `@/` if configured).
